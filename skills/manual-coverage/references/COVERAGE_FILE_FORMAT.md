# Coverage File Format

`coverage.manual.yml` and `coverage.e2e.yml` share the same grammar. Both are read by `@testomatio/reporter run --filter "coverage:file=<path>,diff=<branch>"`.

## Top-level shape

A YAML map. Keys are file paths or globs relative to the repository root. Values are lists of identifiers — Suite IDs, Test IDs, or tags.

```yaml
<file or glob>:
  - "@S<8 chars>"   # Suite ID
  - "@T<8 chars>"   # Test ID
  - "@<tag>"        # Tag (matches anything tagged with that label)
```

## Identifier types

| Identifier      | Format       | Example       | Meaning                                    |
| --------------- | ------------ | ------------- | ------------------------------------------ |
| Suite ID        | `@S` + 8 hex | `@S816410d6`  | All tests inside the suite are selected.   |
| Test ID         | `@T` + 8 hex | `@T6f8e9174`  | Only this specific test is selected.       |
| Tag             | `@<word>`    | `@smoke`      | All tests tagged with the label.           |

IDs are server-generated by Testomat.io. They appear in markdown manual tests inside `<!-- suite ... id: @S... -->` and `<!-- test ... id: @T... -->` blocks, and in automated test code as `@S.../@T...` markers in describe / it / test names.

## File keys

Either an exact path or a glob:

```yaml
# Exact file
app/models/user.rb:
  - "@S816410d6"

# Glob — any file under the directory
app/services/jira/**:
  - "@jira"

# Multiple identifiers per file
app/controllers/users_controller.rb:
  - "@S816410d6"
  - "@T6f8e9174"
```

## Comments

Use `#` to annotate each identifier with its human-readable title or purpose. The reporter ignores them, but they keep the file auditable:

```yaml
app/models/user.rb:
  - "@S816410d6"  # Suite: User permissions
  - "@S1bee6155"  # Suite: Read-only users
```

## Reporter usage

```bash
# Manual run — select cases affected by diff vs main
npx @testomatio/reporter run \
  --kind manual \
  --filter "coverage:file=coverage.manual.yml,diff=main"

# E2E run — pass the actual test runner; --filter generates --grep
npx @testomatio/reporter run "npx playwright test" \
  --filter "coverage:file=coverage.e2e.yml,diff=main"
```

The `diff` value must be a stable branch (e.g. `main`, `origin/main`) the reporter can run `git diff` against.

## Authoring tips

- Prefer Suite IDs when most of a suite relates to a file — it's terser and survives test additions.
- Use Test IDs when only one test in a large suite is relevant.
- Use tags for cross-cutting concerns spanning multiple suites (`@jira`, `@billing`, `@smoke`).
- Avoid empty mappings — a key with `[]` is dead weight.
- Commit the coverage file to the repo so CI and teammates use the same mapping.
